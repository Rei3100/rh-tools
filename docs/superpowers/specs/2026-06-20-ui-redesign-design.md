# Phase 2: UI 全面リニューアル — 設計書

## Context（なぜやるか）

フェーズ1で「読み取り専用ビューア」は完成した。ただし WPF 既定スタイル
そのままで、ユーザーから「**毎日使うソフトとして使いたいが、芋っぽい・
文字が小さい・読みづらい・ライトモード気味で目に痛い**」というフィード
バック。今後の機能拡充（翻訳保持・自動並び替え等）を見据え、UI 全体を
モダンで読みやすいダークテーマに刷新する。

ユーザーは本ソフトを「Reloaded-II 本体に代わるメイン UI」として日々使
う計画なので、見た目品質と使用感はプロダクトの中心要素。

ユーザーは非プログラマー。「優れたデザインのソフトを毎日使いたい」と
明言。

## 目標

- ダーク固定・日本語化・読みやすさ最優先の見た目
- 拡大率を 1% 単位で自由に調整でき、どの倍率でも UI が崩れない
- 将来の機能追加（翻訳・自動並び替え・設定）を素直に足せる構造
- 最小化はタスクトレイへ
- アプリ独自アイコンでタスクバー / トレイ / タイトルバー上で識別可能

## 非目標

- 機能追加（翻訳・自動並び替え・メタデータ補完）はこのフェーズでは扱わない。次フェーズ。
- 複数テーマ・複数フォント等のカスタマイズシステムは作らない。設定で
  変えられるのは拡大率と運用系の数項目だけ。

## ユーザーと合意した方針

- **拡大率のみカスタマイズ可能**。テーマ・フォント・アクセントカラー・
  スクロールバー表示は固定（最良案 1 つだけ）。「自分でフィットさせる
  楽しさ」より「迷わなさ」を優先。
- 細部のデザイン判断は実装者（Claude）が行う。

### MOD 一覧の振る舞い

- 読み込み順を上から表示（既存仕様）。
- `EnabledMods` に含まれるかどうかで ON/OFF トグル位置を決定（既存仕様）。
- 検索は名前・ModId の case-insensitive contains（既存仕様）。
- スクロール：ホイール / タッチパッドは効く。**スクロールバーは常に
  非表示**。
- 大量 MOD（200 件超）の描画パフォーマンス：WPF の `VirtualizingStackPanel`
  / `VirtualizingPanel.IsVirtualizing="True"` を `ItemsControl` でも使え
  るよう `ListBox` 系で実装する。

### ON/OFF トグルの見た目（チェックボックスではなくスイッチ）

- 幅 44px / 高さ 24px / 半径 13px の角丸長方形
- ON：背景 `#4caf50`、丸ノブが右側 3px
- OFF：背景 `#3a3d42`、丸ノブが左側 3px
- 状態はあくまで「現在の読み込み順設定における ON/OFF を可視化」する
  もの。**本フェーズではトグルは表示専用（クリックで切り替えしない）**。
  AppConfig への書き戻しはフェーズ3で扱う。

### スクロールバー非表示の実装

- 全 `ScrollViewer` で `VerticalScrollBarVisibility="Hidden"` 系を既定
  にする ResourceDictionary を定義
- `HorizontalScrollBarVisibility="Disabled"` を基本（横スクロールは封じ、
  テキストは `TextWrapping=Wrap` または `TextTrimming=CharacterEllipsis`
  で対応）

### アプリアイコン

- **緑グラデーション（`#4caf50` → `#2e7d32`、135deg）の角丸正方形**に
  小文字白「**rh**」を中央配置
- 解像度：16 / 24 / 32 / 48 / 64 / 128 / 256 px の `.ico` をひとつ作り、
  EXE のアプリケーションアイコンとして埋め込み（`<ApplicationIcon>`
  in csproj）
- ウィンドウ左上 / タスクバー / トレイ / EXE 自体・すべてで同じ見た目
- 生成は **System.Drawing で実行時生成しない**。実装計画でツール
  （ImageMagick / Inkscape / pwsh + System.Drawing コンソールスクリプト
  等）で **ビルド時 / 開発時に一度だけ書き出して** 静的 `.ico` として
  リポジトリに追加する。

### 最小化動作 — タスクトレイ格納

- 最小化操作（`StateChanged` で `WindowState.Minimized`）を検知したら、
  - `ShowInTaskbar = false`
  - 非表示（`Hide()`）
  - トレイにアイコンを表示（既に表示中なら何もしない）
- トレイアイコンを左クリック / コンテキストメニュー「表示」で復帰：
  - `Show()` → `WindowState = Normal` → `ShowInTaskbar = true` → `Activate()`
- コンテキストメニュー「終了」でアプリ完全終了
- アプリ終了時はトレイアイコンを必ず破棄
- 実装は標準 BCL では足りないので、軽量ライブラリを 1 つ採用：
  - **第一候補：H.NotifyIcon.Wpf**（NuGet）— 小型・WPF 専用・依存少
  - これは「外部ライブラリ禁止」原則の **例外**として認める。理由：
    Windows 10/11 のトレイ API は WPF に標準では生えておらず、自前実装
    すると WinForms 経由のフックや P/Invoke を多用してメンテが重い。
    フェーズ1の「System.Text.Json only」制約は **runtime data parsing**
    が対象で、プラットフォーム機能アダプタは対象外。
- 設定で「最小化したらトレイに格納する」を ON / OFF できる（既定 ON）

### 設定画面（左サイドナビの ⚙）

UI 拡大率の操作と運用系の数項目だけ。シンプルに 1 画面。

- **UI 拡大率**：80%〜200%、1% 刻み
  - スライダー（左右）と数値入力（直接打てる）の両対応
  - 「リセット（100%）」ボタン
- **Reloaded-II フォルダ**：現在の保存パスをテキスト表示、「変更...」
  ボタンでフォルダピッカー、選んだら検証して保存
- **起動時の動作**
  - □ 最小化したらタスクトレイに格納する（既定 ON）
  - □ 最後に開いていたゲームを覚える（既定 ON、Phase1 では未実装だが
    本フェーズで追加）

設定の保存は既存の `SettingsStore`（`%APPDATA%\ReloadedHelper\settings.json`）
を拡張。新しい項目を `AppSettings` に追加するだけ。

### 拡大率の実装方針

- ルート要素（`MainWindow` の直下グリッド）に `LayoutTransform =
  ScaleTransform(scale, scale)` を一度だけ適用
- `scale = UiZoomPercent / 100.0`
- 拡大率変更時はバインドで即時反映（再起動不要）
- 全レイアウトは固定ピクセルではなく `Auto` / `*` / `MinWidth` / `MinHeight`
  中心。固定値が必要な場合も小さめに留め、内容に合わせて伸縮する
- フォントサイズは ResourceDictionary に DynamicResource として定義し、
  `LayoutTransform` が掛かるので追加処理不要
- ウィンドウサイズ自体は `SizeToContent="Manual"` のままで OK（ユーザー
  がウィンドウを広げれば中身は元の比率を保って広がる）
- 拡大率 80% / 100% / 125% / 150% / 200% の 5 点で目視確認することを
  受け入れテストに含める

## アーキテクチャ変更

### MVVM 構造の維持

フェーズ1の `MainViewModel` をベースに、必要な ViewModel を足す形：

- `MainViewModel`（既存）：ゲーム / MOD / 検索状態は据え置き
- `SettingsViewModel`（新規）：拡大率・トレイ設定・「最後のゲーム」設定
  の永続化を担う
- `ShellViewModel`（新規・軽量）：左サイドナビの「MOD 一覧 / 設定 / ヘルプ」
  切替状態だけ持つ

サイドナビでビュー切替するため、メインのコンテンツ領域に `ContentControl`
を置き、`ShellViewModel.CurrentView` に応じて UserControl を差し替える。

### 新規 UserControl

- `Views/ModListView.xaml`：今までの MOD 一覧 + 上部タブ + 検索
- `Views/SettingsView.xaml`：設定画面
- `Views/HelpView.xaml`：「このソフトについて / バージョン / ライセンス
  / GitHub リンク」の極小ヘルプ画面

### MainWindow の再構成

`MainWindow.xaml` は全面書き換え。構造：

```
MainWindow
└─ LayoutTransform (ScaleTransform x:Name=Zoom)
   └─ Grid 2 列 (64 / *)
      ├─ SideNav (64)
      └─ ContentControl Content="{Binding CurrentView}"
```

`MainWindow.xaml.cs` は `StateChanged` ハンドラでトレイ格納処理。

### ResourceDictionary 追加

`App.xaml` の `<Application.Resources>` に色・サイズ・スタイルの
ResourceDictionary を集約。具体的には：

- `Themes/Colors.xaml`：上記カラーテーブルの全エントリを `Color` /
  `SolidColorBrush` で定義
- `Themes/Typography.xaml`：フォントファミリ・サイズ・FontWeight を
  DynamicResource として定義
- `Themes/Controls.xaml`：`Button` / `ToggleButton` / `TextBox` /
  `ScrollViewer` / `ListBox` / `ListBoxItem` の Style と
  `ControlTemplate` を全て定義（ダーク化・スクロールバー非表示）

これら 3 つを `MergedDictionaries` で読み込む。

### 既存 Core の扱い

- `MainViewModel.LoadFrom`、`ModFilter`、`LoadOrderBuilder`、`ModCatalog`
  / `GameCatalog`、`AppSettings`、`SettingsStore` は **既存のまま保持**。
  ロジックは触らない。
- `AppSettings` に新しいプロパティ追加（後方互換、欠落フィールドは
  既定値で読み込む）：
  - `int UiZoomPercent` = 100
  - `bool MinimizeToTray` = true
  - `bool RememberLastGame` = true
  - `string? LastGameId` = null

## 採用する外部ライブラリ

- **H.NotifyIcon.Wpf**（タスクトレイ用、例外として導入）

他は引き続き標準 BCL のみ。Newtonsoft.Json は持ち込まない。

## 受け入れ条件（フェーズ 2 完了基準）

1. アプリ起動直後、左にサイドナビ・上にゲームタブ・中央にカード形式
   の MOD リスト・右に詳細パネルが表示される
2. ゲームタブが日本語表示 (`AppName`) になっている
3. MOD カードに順番・サムネ・名前・作者・スイッチ ON/OFF が表示され、
   スクロールバーが見えない状態でホイールスクロール可能
4. 詳細パネルに依存関係が表示されない
5. URL が無い MOD では URL 行ごと出ない
6. 設定画面で UI 拡大率を 80%〜200% / 1% 刻みで変えると、即座に全体
   サイズが変わり、80% / 100% / 125% / 150% / 200% で UI が崩れない
   （要素重なり・はみ出し・テキスト切れ無し）
7. ウィンドウ最小化で、タスクバーからアイコンが消え、トレイにアイコン
   が出現する。トレイクリックで復帰する。トレイ右クリックで「表示」
   「終了」メニューが出る
8. EXE 自身・ウィンドウ・タスクバー・トレイで「rh」ロゴアイコンが表示
   される
9. 全テキストが日本語、`true` / `false` がどこにも出ない
10. フェーズ1 までの単体テストが引き続き全件 PASS（ロジック層は不変）
11. GitHub Actions が緑、`ReloadedHelper.App.exe` が成果物として
    生成される

## 既知の非対応 / 次フェーズに送る項目

- ON/OFF トグルでの **実際の有効化操作**（クリックで Reloaded-II 側の
  AppConfig を書き換える）はフェーズ 3 で扱う
- 翻訳・自動並び替え・メタデータ補完は別フェーズ
- ヘルプ画面はバージョン / GitHub リンクのみの最小実装
- 設定画面の「テーマ切替」「フォント切替」は意図的に作らない
